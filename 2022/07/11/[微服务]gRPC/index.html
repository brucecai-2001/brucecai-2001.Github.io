微服务(2)---RPC和gRPC

Author: Xinyu Date: July 11, 2022  17:46:43 Category: 微服务

RPC协议

像调用本地方法一样调用远程的过程。

RPC采用客户端/服务端的模式，通过request-response消息模式实现(http)

gRPC是一个RPC框架，客户端可以像调用本地对象一样直接调用另一台不同的机器上服务端应用的方法。服务端实现这个接口，并运行一个gRPC服务器来处理客户端调用。


gRPC

gRPC负责处理服务与服务之间的调用。

数据在网络中传输需要进行序列化，例如xml, json等, gRPC使用protocol buffer, 这是谷歌的数据结构序列化机制。

gRPC底层协议: http2




gRPC将元数据放在http2/header里，参数序列化后放到http2/Data中


protocol buffer

谷歌开源, 高性能，适合对响应速度有要求的数据传输场景。需要序列化和反序列化。

  1. 体积小
  2. 跨平台多语言
  3. 兼容性强
  4. 序列化，反序列化速度快

一个proto文件


protoc 序列化和反序列化


生产gRPC

有可能出错的地方

“google.golang.org/grpc” 不是最新版本

go install google.golang.org/protobuf/cmd/protoc-gen-go@latest

go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest

export PATH=”$PATH:$(go env GOPATH)/bin”
